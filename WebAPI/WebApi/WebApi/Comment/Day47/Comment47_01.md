github에서 repo를 만들면
readme.md 파일이 만들어집니다.

md => mark down

[MarkDown]
 => text로만 이루어진 글꼴효과/크기 이런것들을 줄수있는 문법

 간단하고 뚜렷하게 효과를 줄수있음.


Q. 문서작업할때 그냥, md 말고 word쓰면 되지않을까요?
A. 워드를 쓰면, 코드 스니핏을 쓸수도 없고, 무엇보다도 중요할때 어따 적었지? 할때가 되게 많아요.

Q. 어? 그러면 그냥 메모장에 적으면 되지 않을까요?
A. 표, 제목, 링크 최소한의 문법으로 간단하게 이쁘게 그릴수있는 (문서는 가시성도 중요해서) md 문법입니다.



Visual Studio Code
 => 얘는 메모장기능으로 만들어진 IDE입니다.
 근데, Extensions을 사용자가 스스로 추가할수있게 만든 메모장입니다.


MD 파일은 읽는곳마다 살짝씩 다릅니다.
#, ##, ### 을 제목으로 칭하겠다.
