[서버]
  - Client의 요청을 받아서, 응답을 해주는 역할을 함.

  - PC를 말할 때도 있고
  - Program을 말할 때도 있고
  - 추상적인 의미를 내포하고 있을 때도 있음

[Web Server의 동작 방식] (코드적인 부분)
 - Programs.cs -> 실행점 / 초기화 코드
 - Controllers/WeatherForecastCntroller.cs
      -> Web의 아키텍처에는 Model - View - Controller
      -> Model : (화면에 보이는 데이터)
      -> View : (화면 구성), (View에서 Model을 가져다 씀)
      -> Controller : (기능), (어떤 데이터를 어떤 View에 넘길지), (핵심)

    - 기본적으로 Controller는 ControllerBase를 상속받고 있음
    - Controller 네이밍 규칙은 
    [WeatherForecast][Controller]

도메인 + Controller

도메인 -> 아까 인프라에서의 도메인 = IP
도메인 -> 핵심 데이터 (User, Auth, Item, Hero, Character)

Web 통신은 기본적으로 json 형식으로 통신을 하게 됩니다.
